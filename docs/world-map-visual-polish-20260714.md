# ワールドマップ・ビジュアル強化 2026-07-14

## 不具合修正

- ワールドマップで、主人公を囲むように大きな四角い明暗領域が表示される問題を修正した。
- 原因は固定マップ向けの加算合成照明を、ズームされたCanvas描画のワールドマップにも適用していたこと。
- `WORLD` では画面固定の雰囲気レイヤーを生成しない。ワールドの情報量は地形に固定された透過デコで増やす。

## 生成・配置した素材

- 草原
  - 花と小石
  - 穂草と雑草
  - 露出した土、苔、小石
- 森林
  - シダ、落葉、切り株、根
  - 根、苔石、落葉
- 低山
  - 岩塊、乾いた土、砂利
- 海岸
  - 水タイルと陸タイルの境界に沿う白波

## 配置規則

- 座標ハッシュで素材、左右反転、微小角度を決定する。同じ座標では毎回同じ外観になる。
- 草原は3種類、森林は2種類を混在させる。
- 海岸の白波は上下左右の隣接地形を調べ、陸側へ向けて回転する。
- 装飾は描画専用。移動判定、エンカウント、イベント座標、ストーリーフラグ、マップデータを変更しない。

## キャッシュ

- ワールド進入直後の欠落を防ぐため、今回のワールド装飾7点を初期重要画像へ追加した。
- 当時のキャッシュ識別子は `v3.67-world-polish`。2026-07-15の初回全量キャッシュ復元、指定橋、ダンジョン暗幕補正、終盤固有戦闘エフェクト追加を経て、生成後到達性・縦横マップ表示・単一確率表・水面限定の船表示・水没専用戦闘背景・ランダム外観再開復元対応後、実プレイ再開経路の二重初期化を除去。全固定・ランダムダンジョンの壁面を共通レジストリ化しつつ、禁忌の森は単一壁面化せず既存Wチップ4種類のランダム配置へ復帰した現行は `v3.83-forest-wall-variants`。実装時は `main.js` / `sw.js` / `assets.js` の現行値を正とする。

## 画像生成方式

- built-in image generationを使用。
- 平坦なマゼンタ背景で生成後、`remove_chroma_key.py` で透過化した。
- `tools/finalize-generated-overlay.py` で余白、縦横比、縮小、透過隅を検証した。
- 生成元は `assets/managed/source/generated/20260714-world-map-polish/` に保存する。
