# ガルヴァニアへの洞窟 全面刷新 実装レポート

## 対象

今回の変更対象は、ガルヴァニアへの洞窟と、それを成立させるための最小限のロジックのみです。

- `map.js`
  - `GALVANIA_CAVE_NORTH` / `GALVANIA_CAVE_SOUTH` を、同一内容・同一進行キーの6階層ダンジョンへ刷新
  - 北口は1階、南口は6階に接続
  - 北口のみ `lightPalaceCleared` の入場制限を維持
- `dungeon.js`
  - 固定ダンジョンの開始階指定に対応
  - 北口・南口を同一進行キーで扱うための鍵スコープ調整
  - 固定ダンジョン用の限定マップ表示ロジックを追加
- `maps_logic.js`
  - `canonicalAreaKey` による固定ダンジョン進行キー共有に対応
- `main.js`
  - 固定ダンジョンの限定マップ表示をミニマップに反映

## 設計方針

ロンダルキアへの洞窟系の「落とし穴・無限ループ・探索ミスの消耗感」と、ネクロゴンドへの洞窟系の「火山・溶岩・最終目的地へ抜ける通過儀礼感」を、既存ゲームの固定ダンジョン仕様に合わせて再構成した。

ただし、この洞窟は魔王城へ向かう通り道であるため、必須ボスは置いていない。中ボスは以下の役割に限定した。

- レア宝箱の横穴を守る
- ショートカット用の金鍵を持つ
- 通過しなくてもクリア可能

## 6階層構成

| 階層 | 名称 | 主な狙い |
|---|---|---|
| 1階 | 北口 黒岩の胎道 | 北口導入。倒れた兵、黒岩の道標、魔族斥候ハンター |
| 2階 | 偽りの無限回廊 | ロンダルキア風の無限ループ。石碑のヒントで正解ルートを示す |
| 3階 | 溶岩の地底湖 | 溶岩床、熱ダメージ、限定マップ表示、任意の黒炎番人 |
| 4階 | 氷晶の十字滑床 | 十字路の氷床。曲がれない／遠回りさせる配置。任意ボスが金鍵を所持 |
| 5階 | 魔軍補給路 | 魔王城への補給路。金鍵ショートカット、徘徊する上級魔族、限定マップ表示 |
| 6階 | 南口 白骨の旧坑 | 南口への最終導線。出口付近の溶岩割れ目と、任意の宝箱守護魔 |

## 宝箱・中ボス

宝箱は合計26個。

- 1階：4個
- 2階：4個
- 3階：4個、任意ボス1体
- 4階：4個、任意ボス1体、金鍵報酬
- 5階：5個、金扉ショートカット
- 6階：5個、任意ボス1体

## 追加ロジック

### 固定ダンジョン開始階

`entryPoints` に `floor` を持たせ、南口から開始した場合は6階から入れるようにした。通常プレイでは北口から1階へ入る。

### 同一進行キー

北口キーと南口キーは残しつつ、`canonicalAreaKey: "GALVANIA_CAVE"` を持たせた。これにより、宝箱開封、ボス撃破、マップ変化、限定マップ表示の進行キーを共有する。

### 限定マップ表示

3階と5階は `limitedMapReveal: true`。ミニマップ上では、未踏破部分を壁として扱い、歩いた範囲だけが永続表示される。

## 検証

実施済み。

- `node -c map.js` 成功
- `node -c maps_logic.js` 成功
- `node -c dungeon.js` 成功
- `node -c main.js` 成功
- 独自到達検証で、全6階の入口、階段、出口、宝箱隣接マス、石碑・倒れた人、任意ボス座標の到達を確認
- 通常ルートは任意ボスを倒さなくても北口から南口まで到達可能
- 氷床フロアは実移動ルールに近い滑走処理で到達検証済み

既存の `tools/validate-map-safety.js` は、今回の修正前から `MapRegistry is not defined` で停止する読み込み順問題があるため、今回のガルヴァニア検証には独自検証を使用した。
