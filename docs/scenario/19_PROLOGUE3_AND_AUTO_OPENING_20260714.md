# PROLOGUE3接続・自動進行OP 実装稿 2026-07-14

## 承認根拠

ユーザーから、紙芝居OPの発生地点を `PROLOGUE3` 終了時へ変更し、長老の終盤会話とOP本文を現行ストーリー全体に合わせて直接改稿する指示を受けた。新規背景画、透明度の高い字幕、一文ずつの表示、自動進行も同時に実装する。

## 発生順

1. 初回起動時に全画像を今ダウンロードするか確認する。「はい」は完了を待ち、「いいえ」は即開始して全画像をバックグラウンドでキャッシュする。`start_adventure` は従来どおり、村の依頼、ガイル・サラ加入、`storyStep: 1 / subStep: 1` までを処理し、ここではOPを出さない。
2. 始まりの洞穴を攻略して `storyStep: 2 / subStep: 0` となり、長老へ報告すると `PROLOGUE3` が始まる。
3. `PROLOGUE3` 終了後、`subStep: 1` へ進め、紙芝居OPを再生する。
4. OP終了後に全画像取得案内を重複表示しない。
5. 旧発火位置の再生済み状態と衝突しない `openingPrologue3Viewed` により再生済み状態を保存する。

## 初回起動時の全画像キャッシュ

- 初回起動時に全画像を前景で取得するか確認し、「はい」では全量取得完了を待つ。「いいえ」でもゲーム開始後に全量バックグラウンド取得を開始する。
- ニューゲーム開始前のメモリ先読みには、リュミナ村と開幕ジェリー戦だけでなく、始まりの洞穴で遭遇する通常敵 `100002`、`100003`、`100004`、ボス `301000` を含める。
- フィールド戦闘背景 `battle-field-ai.png` と洞窟戦闘背景 `battle-dungeon-ai.png` の両方を取得し、ゲーム開始時の画像ロード対象にも入れる。
- 洞穴の壁・床マップチップと、マップ上に表示するボス画像もゲーム開始時の画像ロード対象に入れる。
- ここまでを「即時メモリ先読み対象」とし、それ以外も `assets.js` の全量キャッシュ対象から除外しない。画像のオンデマンド取得を基本方針にしない。

## OP字幕（一文一画面・自動進行）

後半地域へ到達して初めて知る異変を先取りしない。プレイヤーが設定した主人公名や、序盤の村名・村内での細かな出来事も語らず、「滅びた故郷・炎の異変・正体不明の不穏な広がり」だけを扱う。

1. あの日、王都では「大いなる祝福」の儀式が行われた。
2. 六つの力を結ぶはずだったプリズマは、白く瞬いたあと、音もなく割れた。
3. 七色の破片は空へ散り、地の底には、地図にない裂け目が走った。
4. 同じ夜、遠い村がひとつ、冷たい風と底知れぬ闇に呑まれた。
5. 生き残った者は、答えのないまま剣を取った。
6. やがて、地の底から、あの夜と同じ風が吹き始める。
7. 東では、炉の火までもが人の手を離れようとしていた。
8. 異変はまだ、世界の片隅で静かに息をしている。
9. その始まりを確かめるため、旅は続く。
10. 字幕を消し、タイトル画面と同じ `assets/background/PRISMA ABYSS.png` を表示する。

## 映像・UI仕様

- 全字幕で新規画像 `assets/generated/opening-prism-collapse-v002.png` を使用する。最初に生成した高密度都市版v001は世界観不一致として破棄し、実装では参照しない。
- 木造家屋、畑、川、小さな石造城、祠が点在する前工業的な世界規模を守る。巨大魔法都市、未来的建築、過剰な装飾は使わない。
- SFC後期からPS2初期のゲーム内イベント画・取扱説明書を思わせる、手描きガッシュ／水彩調の落ち着いた塗りを基準にする。
- 一枚絵の左・中央・右・上・下をページごとにパンし、拡大率と移動方向を変える。
- OP開始時はフィールドから直接一枚絵へ切り替えず、黒へのフェード、短い完全暗転、OP背景のフェード表示を挟む。
- 文章は一文だけを中央下寄りへフェードインし、一定時間保持後にフェードアウトする。
- 背景パネルは置かず、文字の直下にごく薄いグラデーションと文字影だけを使う。
- 字幕は画面最下部へ寄せず、下端から約14～18vh上へ置く。
- 最終ページは文字でタイトルを組まず、`main.html` と同じタイトルロゴ画像を中央表示する。
- 操作なしで次の文へ進む。クリック、Enter、Spaceでは現在文を送れる。
- `SKIP` は残す。最後はタイトルを短く表示して自動終了する。
- `prefers-reduced-motion` ではパン・揺れを抑えるが、自動進行は維持する。
