# 01_WRITING_RULES

PRISMA ABYSSの文体・会話・演出ルール。

## 目的

AIっぽい文章を避け、キャラクター・土地・生活・誤解・沈黙・断片から物語を立ち上げる。

## セリフ行ルール

- セリフウィンドウ1行は原則30文字程度まで。
- 長い説明は複数行に分ける。
- 1キャラが連続して複数回話すのは可。
- `\n` で区切られた各行も30文字程度に収める。
- システム文も長くしすぎない。

悪い例:

```text
この神殿には古代のプリズムを守るために作られた結界があり、王国軍はそれを利用して世界を支配しようとしているのです。
```

良い方向:

```text
壁の文字が削れている。
「……リズムを、守れ」

その先は読めない。

兵士の靴跡だけが、
新しく残っている。
```

## AIっぽさ禁止

避けるもの:

- 全員が同じ丁寧語で話す
- 会話が説明書になっている
- 感情を抽象語で説明する
- 「真実」「運命」「世界の理」などを多用する
- NPCが攻略情報だけを話す
- 古文書が真相を全部語る
- 伏線を置いた直後に意味を説明する
- 一文のリズムが綺麗すぎる
- 選択肢やイベントが都合よく話を進めすぎる

使いたいもの:

- 言い淀み
- 誤解
- 噂
- 方言や土地の言い回し
- 仕事の都合
- 家族の心配
- 古い傷
- 子どもの勘違い
- 商人の損得
- 兵士の疲労
- 神官の口癖
- 手紙の消し跡
- 破れた看板
- 煤けた石碑

## 情報開示ルール

キャラは、その時点で知らないことを話してはならない。

ただし、以下は積極的に使ってよい。

- 個人的な憶測
- 間違った噂
- 恐怖による決めつけ
- 家族から聞いた古い話
- 兵士の建前
- 神官の教義
- 商人の損得込みの嘘
- 子どもの聞き間違い

重要:

キャラの誤解は、後で成長や反省につなげてよい。
作者の説明不足や矛盾を、キャラの誤解でごまかしてはいけない。

## 伏線ルール

伏線は「今わかる説明」ではなく「後から気づく違和感」にする。

よい伏線:

- その場ではただの生活会話に見える。
- キャラの感情と結びついている。
- 後半で別の意味に変わる。
- 量が少ない。
- 読み返すと気づく。

悪い伏線:

- 明らかに意味深すぎる。
- 何度も同じ表現で繰り返す。
- NPCが作者の代弁をする。
- 古文書が答えを言う。
- 直後に説明される。

## 伏線の密度

1エリアあたりの目安:

- 大きな伏線: 0〜1個
- 小さな違和感: 2〜5個
- ミスリード: 1〜3個
- 生活会話: 多め

プレイヤーが「また伏線か」と感じたら多すぎる。

## NPC会話ルール

NPCは世界に住んでいる人間として書く。

NPCが話してよいこと:

- 今日の仕事
- 家の修理
- 子どもの熱
- 鍛冶場の注文
- 魚や作物の値段
- 昨日の喧嘩
- 祭りの準備
- 古い迷信
- 旅人への警戒
- 兵士への不満
- 魔王軍への恐怖
- 王国軍への信頼または疑い
- どうでもいい自慢

NPCが毎回話してはいけないこと:

- 次の目的地
- ダンジョン攻略法
- 世界の真相
- プリズムの設定説明

## 遺跡・書物・碑文ルール

遺跡や書物は、説明媒体ではなく、欠けた証拠として扱う。

使える表現:

- 文字が欠けている
- 墨がにじんでいる
- 後世の書き足しがある
- 王国暦と地方暦が違う
- 読みが複数ある
- 神官が一部を削った痕跡
- 子どもが落書きしている
- 石碑が上下逆に置かれている

## 既存会話レビュー方針

既存会話に問題を見つけても、即修正しない。

やること:

1. `07_DIALOGUE_REVIEW_QUEUE.md` に記録する。
2. 現行文を残す。
3. 改善案を複数出す。
4. Codexの推奨を添える。
5. ユーザー承認を待つ。

やってはいけないこと:

- 現行文を自動的に正本扱いする。
- 現行文を勝手に削除・改変する。
- 改善案を正本に混ぜる。
- ユーザー判断欄を勝手に埋める。
