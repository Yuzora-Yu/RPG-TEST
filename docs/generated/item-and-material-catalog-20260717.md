# 道具・素材拡張 実装台帳（2026-07-17）

## 正本とID帯

- 既存道具: `items.js`
- 追加道具・素材: `item-expansion.js`
- 戦闘／非戦闘共通処理: `item_runtime.js`
- 通常敵ドロップ方針: `monster-drop-policy.js`
- 追加戦闘道具: ID 1000–1079内の64件
- 素材: ID 2000–2063の64件

追加データを既存の巨大配列へ散在させず、用途別ファイルをロード順で結合する。ID重複時は追加せず、検証スクリプトで総数と重複を検査する。

## 戦闘・回復・補助道具

- 属性攻撃35件: 火・水・雷・風・光・闇・混沌 × 単体弱／単体強／全体弱／全体強／ランダム3回強
- 全体MP回復4件: 30／100／200／999
- 全体HP回復2件: 80／350
- 通常強化10件: 攻撃・防御・魔力・魔法防御・全属性耐性 × 単体50%／全体30%、4ターン
- キャンプ1件: 非戦闘専用、全員蘇生・HP/MP全回復
- 超強化2件: 会心率／回避率を単体30%、2ターン
- 弱体10件: 攻撃・防御・魔力・魔法防御・全属性耐性 × 単体50%／全体30%、4ターン

Rank、価格、店頭可否、メダル限定可否は各道具定義に保持する。道具屋は店Rank以下のみ、深淵宝箱は階層Rank以下を「道具65%・素材35%」で抽選する。メダル限定品は深淵宝箱と道具屋から除外する。

## 素材

金属・木材・魔石・羽・爪・毛皮・液体・分類不能の8分類に、G/F/E/D/C/B/A/Sの8段階を用意した。現段階では売却と所持のみで、生成・加工用途は後続実装とする。

## ドロップ方針

- レア雑魚4体: 通常ドロップを「ちいさなメダル」100%に統一。レア枠は育成道具。
- 通常雑魚196体: 通常枠を素材94体、回復道具73体、攻撃・強化・弱体道具29体へ配分。素材率は約48%。
- 序盤Rank 25以下: 素材を約1/3に抑え、非素材枠は薬草系・MP回復系を優先。
- 通常枠が素材ならレア枠を回復／戦闘道具、通常枠が道具ならレア枠を1段階上の素材とする。
- 各Rank帯の二回行動エリート20体: レア枠を育成道具ID 100～106に固定し、転生の実ID 107は除外。
- ボス・特殊ボス: 既存の個別ドロップ定義を維持し、この一括方針の対象外。

## アイコン

メニュー上の視認性を優先し、個別道具ではなくカテゴリ共用の4種類だけを使う。

- 攻撃: 赤い爆弾
- 強化: 青い薬瓶＋上矢印
- 弱体: 紫の薬瓶＋下矢印
- 素材: 素材袋

生成原画・透過中間画像・64pxランタイム画像・再生成スクリプトは `assets/ui/menu-icons/source/item-category-v001/` と `tools/assets/build-item-category-icons.py` で管理する。

## 検証

`tools/validation/validate-item-expansion.js` が、件数、7属性×5形式、素材8×8、戦闘／非戦闘効果、深淵抽選、店・メダル導線、全雑魚ドロップ参照、アイコン寸法・透過・キャッシュ登録を一括検査する。
