# 戦闘ロジック検証仕様

更新日: 2026-07-17

## 目的

特技、戦闘道具、作戦別オート戦闘、敵の行動決定、状態異常、バフ・デバフの変更時に、実データと実行ロジックの不整合を自動検出する。

## 実行方法

個別監査:

```powershell
node tools/battle/battle-logic-audit.js
```

JSON出力:

```powershell
node tools/battle/battle-logic-audit.js --json
```

全検証への組み込み:

```powershell
node tools/validation/run-all.js
```

`tools/validation/validate-battle-logic.js` は全検証から自動実行される。

## 現在の設計仕様

### スキル使用設定

- `hiddenSkills`: 手動戦闘メニューと非戦闘時スキルメニューの表示／非表示を管理する。
- `autoDisabledSkills`: オート戦闘での使用可／否を管理する。
- 両設定は独立している。手動では使うがオートでは使わない設定が可能。
- 旧セーブの `hiddenSkills` は、従来と同じ動作を保つため初回移行時に `autoDisabledSkills` にも複製する。
- 設定形式は `skillUsageConfigVersion: 2` とする。

### オート戦闘

- ガンガンいこうぜ: 攻撃を優先し、低HP時のみ最低限の回復を行う。
- バッチリがんばれ: 蘇生、回復、状態回復、弱体解除、敵強化解除、攻撃の順で状況判断する。
- せつやくしようぜ: 通常攻撃で倒せる敵、残MP、消費MP、与ダメージ効率を考慮する。
- いろいろやろうぜ: 弱体と強化を高めの確率で選択し、攻撃も行う。
- いのちだいじに: HP70%以下から回復候補とし、蘇生・状態回復・弱体解除を優先する。
- ＭＰつかうな: MP消費技を選ばず通常攻撃を行う。
- 呪文封印、特技封印、回復封印、MP不足、オート使用否を候補選定時に適用する。

### 敵AI

- 行動条件0は常時、1はHP50%以上、2はHP50%以下、3は状態異常または弱体中。
- MP不足と各種封印を行動候補へ反映する。
- 蘇生対象がいない場合は蘇生技を選ばない。
- 通常攻撃と攻撃技をやや優先し、強化・弱体の連発を抑制する重みを使用する。
- `actCount` は1ターンの行動回数を表す。

### 状態異常・持続効果

- 毒は行動後に最大HPの5%、猛毒は10%、感電は15%のダメージ。
- 同時に存在する場合は感電、猛毒、毒の優先順で1種類だけ行動後ダメージを受ける。
- バフ、デバフ、状態異常はラウンド終了時に残りターンを1減らす。
- `turns: null` の永続効果は自動解除しない。
- 能力倍率バフ・デバフは戦闘中ステータス計算へ反映する。
- 全属性耐性上昇・低下は属性ダメージ計算へ反映する。
