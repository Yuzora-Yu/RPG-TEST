# マップ／戦闘演出ブラッシュアップ 2026-07-14

## 到達目標

- 後期16bit JRPGらしい、少ない解像度の中で材質・高低差・空気感が読める画面を目指す。
- 特定作品の画像・構図・固有デザインは複製せず、手打ち感のある輪郭、限定色、視線誘導、演出の緩急をPRISMA ABYSS独自の素材として作る。
- マップ座標、当たり判定、イベント発火位置は変更しない。装飾と大気演出は完全に非衝突の描画層とする。

## マップ第1パス

- 生成素材 `overlay_decor_grass_wildflowers_v001.png` を草地系マップへ低密度で決定論的に配置する。
- 生成素材 `overlay_decor_stone_cracks_v001.png` を洞窟・固定ダンジョンの通常床へ低密度で配置する。
- 同じ座標では常に同じ装飾を選び、移動のたびに景色がちらつかないようにする。
- 宝箱、階段、イベント、ボス、床効果、NPCのマスには装飾を置かない。
- 地域キーから大気色を選び、炎・水氷・風森・光・闇・深淵を弱い色調と粒子で区別する。
- 装飾画像が未取得でも既存地形だけで安全に描画できること。

## 戦闘エフェクト第1パス

スキルデータから次を判定し、画像、サイズ、残光、粒子量、画面フラッシュを組み合わせる。

1. 種別: 物理、魔法、ブレス、回復、強化、弱体、特殊
2. 属性: 火、水／氷、雷、風、光、闇、混沌、無属性
3. 範囲: 単体、全体、ランダム、自分
4. 威力: `rate`、`base`、消費MP、固定・割合ダメージ、奥義IDから4段階
5. ヒット数: 単発、多段、連続大技
6. 追加性質: 会心、防御無視、毒、即死、回復、蘇生

物理属性攻撃には生成素材 `fx_phys_elemental_arc_v001.png` を使用し、属性ごとの色調と発光を重ねる。魔法・ブレスは既存の属性専用画像を維持しながら、威力段階と範囲によって拡大率・残光・粒子・フラッシュを変える。

## 画像生成記録

- モード: built-in image generation + クロマキー除去
- 原本: `assets/managed/source/generated/20260714-visual-polish/`
- 透過処理: imagegenスキル同梱 `remove_chroma_key.py`
- 仕上げ: `tools/finalize-generated-overlay.py`
- 実行時素材は元画像を上書きせず、新しい `v001` 名で追加する。

## 受け入れ条件

- 装飾は移動、当たり判定、アクション表示、ミニマップへ影響しない。
- 始まりの村と始まりの洞穴で装飾画像が初回から欠けない。
- 全スキルに最低1つの解決可能なエフェクト種別が割り当たる。
- 属性物理が無属性斬撃画像へ黙って退避しない。
- 同じ属性でも低威力単体と極大全体で、画面上の規模と残光が明確に異なる。
- 既存の全体攻撃の左から順にダメージを出す仕様と、戦闘速度設定を維持する。

## 2026-07-15 第2パス

### ワールドマップ

- 海面中央へ波線を置く方式を廃止し、水タイルの上下左右を調べて、陸と接する辺だけへ細い白波を密着表示する。角では二辺を描く。
- 橋の自動推測は使用しない。ワールド座標 `(53,69)`、`(78,58)`、`(79,58)` の3マスだけを海タイルへ変更し、`WORLD_BRIDGES` に登録された海マスだけを歩行可能な `overlay_world_bridge_wood_v001.png` として描く。
- 通常の町・村・固定フィールドでは、画面全体の暗幕、ビネット、主人公中心の加算光、固定画面粒子を生成しない。大気暗転はダンジョンだけに限定する。
- 静的タイル層の署名を4マス単位へ変更し、歩くたびの全GameObject破棄・再生成を避ける。4マスの描画余白を同時に持ち、再利用中もプレイヤー、カメラ、水面、HUDミニマップは毎歩更新する。

### 戦闘エフェクト

- 斬撃 `fx_phys_slash_arc_v002.png`、刺突 `fx_phys_pierce_lance_v002.png`、粉砕 `fx_phys_smash_impact_v002.png`、射撃 `fx_phys_ranged_volley_v001.png` を分離する。
- 弓・矢・射撃・bow・arrow・shotを物理種別判定へ追加し、多段射撃も斬撃コンボへ落とさず射撃画像を使う。
- 回復 `fx_support_heal_radiance_v002.png`、弱体 `fx_support_debuff_hex_v002.png`、ブレス共通形状 `fx_breath_cone_master_v001.png` を追加する。
- 属性ブレスは共通の扇形シルエットへ属性色を適用する。高位属性魔法は既存の属性画像を維持し、その上へ `fx_magic_arcane_burst_v001.png` を回転する副層として重ねる。
- 画像だけで差を付けず、刺突の横移動、粉砕の急拡大、射撃の斜行、ブレスの横展開、高位魔法陣の逆回転を個別キーフレームとして持つ。

### 画像生成・透過記録

- モード: built-in image generation / built-in image edit / クロマキー除去。
- 原本: `assets/managed/source/generated/20260715-bridge-battle-polish/`。
- 初回生成の緑背景に暗い色勾配が残った魔法陣とブレスは、そのまま採用せず、均一マゼンタ背景へ画像編集して再生成した。
- `tools/finalize-generated-overlay.py --alpha-floor` で低アルファの背景ノイズを落とし、四辺の最大アルファ0を確認してから実行素材へ配置した。
- 追加素材はすべて `PRISMA_ASSETS.graphics` または `PRISMA_ASSETS.battleFx` に登録し、初回全量キャッシュの自動構築対象へ含める。

### 実画面検証

- ワールド座標 `(59, 64)` で海岸中央の縦波がないこと、近傍 `(53, 69)` の川渡り地形へ橋が表示されることを確認した。
- `START_VILLAGE` で全面暗幕、ビネット、主人公中心光が出ないことを確認した。
- 斬撃、刺突、粉砕、射撃、回復、高位火魔法＋魔法陣、火属性全体ブレスを実DOM・実CSSで同時再生し、透過矩形が出ないことを確認した。

## 2026-07-15 第3パス

### 描画・大気演出

- 北端の座標 `(2,2)` は、南北ループ境界の外側座標 `(2,92)` から描かれる場合も正規化して同じ施設画像を取得する。
- 翼の8方向・歩行フレームを起動時画像セットへ追加し、Phaser登録が一瞬遅れた場合も直前の主人公画像または通常歩行画像へ退避する。白丸は生成しない。
- ダンジョンの暗幕はカメラの論理表示サイズに上下左右1マスの余白を加えて描き、周囲だけ暗幕が欠ける四角い枠をなくす。中央光は維持する。
- 白い大気粒子はプレイヤー座標を乱数種に使わず、エリア・階層ごとに固定配置し、移動とは独立した3秒周期の往復アニメーションで漂わせる。
- フィールド移動時に `msg-text` の文字だけを空にする旧処理を削除し、宝箱・調査・イベントの最新ログを次のログまで保持する。透過ログ枠だけが残る状態を作らない。
- ランダムダンジョンの移動冒険者は通常NPCと同じ隣接会話へ変更する。冒険者のマスは通行不可とし、会話可能距離では冒険者側も歩き去らない。

### 終盤固有演出

- ID `166/168/238/242/243/244/245/246/247/248` の10技へ、それぞれ別画像・別CSSキーフレームを割り当てた。
- ラグナロクは5連斬、プリズマエンドは単発の防御無視切断、ビッグバンは全体膨張、アビスウォールは下からの壁面上昇、フェニックスフレアは縦落下として動かす。
- 創世の魔力は術式再構築、カオスショックは収縮後の破裂、イルミナティブレイクは3本の光柱、ジ・エンドは左右収束、ロストプリズマは色彩を失う内向き崩壊として動かす。
- 生成原本は `assets/managed/source/generated/20260715-ultimate-skill-fx/`、実行時の透過済み素材は `assets/effect/fx_ultimate_*_v001.png` に保存し、中央の全量キャッシュ対象へ登録した。
