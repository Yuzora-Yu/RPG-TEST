# 07_Codex向け_実装指示メモ

## 目的

現在のゲームを、機能が最初から並ぶ構造から、ストーリー進行に応じて仲間・施設・移動手段・やり込み要素が解放される構造へ整理する。

## 最初にやること

### 1. `progress.unlocked` の拡張

既存セーブ互換を保ったまま、以下のようなキーを追加する。

```js
progress.unlocked = {
  smith: false,
  gacha: false,
  abyss: false,
  teleport: false,
  casino: false,
  medalKing: false,
  boat: false,
  wing: false,
  fixedDungeonEndless: false
}
```

`App.init()` のマイグレーションで存在しないキーを補完すること。

### 2. 序盤ストーリーの整合性修正

アルスは始まりの村出身にしない。彼はすでに別の故郷を失っている旅人。始まりの村は最初に救う村。

既存のガイル/サラ加入は維持しつつ、会話のニュアンスを「村の子たちが、村を救ったアルスに同行する」へ寄せる。

### 3. ストーリー加入キャラの固定化

本編中はガチャを解放せず、以下の順で加入。

```text
始まりの村: ガイル、サラ
炎の里: シャオ
風の集落: エリーゼ
水上都市: ケイト
雷の要塞: ジョセフ
光の宮殿: レイラ
魔王城到着後イベント戦闘: シャニー
```

その他キャラは、NPC/助っ人/噂/イベント登場として扱い、魔王討伐後のガチャややり込みで正式加入させる。

### 4. 機能解放

```text
炎の里攻略 → 鍛冶屋OPEN
水上都市攻略 → 魔法の小舟入手、カジノOPEN
雷の要塞攻略 → メダル王OPEN
魔王城攻略 → ガチャOPEN
深淵初探索 → ダンジョン突入機能OPEN
深淵40階 → 表エンド、光の宮殿イベント、転生の実1個
光の宮殿イベント後 → 41階以降、宿屋転送、属性エンドレス探索
深淵100階 → 光の翼OPEN
```

### 5. `characters.js` に将来的に追加したい項目

イベント実装を楽にするため、キャラデータに `story` メタ情報を足す方針。

```js
story: {
  originArea: "FIRE_VILLAGE",
  joinTiming: "after_fire_village_clear",
  storyRole: "story_join",
  relationshipTags: [
    { target: 306, type: "sister", feeling: "hatred_misunderstanding" }
  ],
  spoilerLevel: 2
}
```

この追加は必須ではないが、今後NPC会話・加入イベント・図鑑・ガチャ演出を作る上で強く推奨。

## 注意すること

- アーカイブ文は非常に濃いので、序盤の会話で全部説明しない。
- SSR/UR/EXキャラは序盤から正式加入させない。噂や回想、助っ人に留める。
- ゼノンは善人化しすぎない。冷酷だが約束を破らない魔王。
- リュシオンは直接登場を遅らせる。序盤は夢、声、光、謎の回復として見せる。
- シャオとシャニーの真相は魔王城まで引っ張る。
- 光の宮殿は「正しい光」ではなく「正しさの危うさ」を描く。
